## ADDED Requirements

### Requirement: 表单包含所有必要字段
发布表单 SHALL 包含以下字段：拼车类型、出发地、目的地、出发时间、座位数（条件显示）、价格（可选）、备注（可选）。

#### Scenario: 显示完整表单
- **WHEN** 用户进入发布页面
- **THEN** 系统显示包含所有字段的表单

#### Scenario: 座位数字段条件显示
- **WHEN** 用户选择"找乘客"类型
- **THEN** 系统显示座位数输入字段
- **WHEN** 用户选择"找车"类型
- **THEN** 系统隐藏座位数输入字段

### Requirement: 必填字段验证
系统 SHALL 验证必填字段（拼车类型、出发地、目的地、出发时间）不能为空。

#### Scenario: 提交时缺少必填字段
- **WHEN** 用户未填写必填字段就点击发布按钮
- **THEN** 系统显示错误提示"请填写必填字段"
- **THEN** 系统不提交表单

#### Scenario: 找乘客模式缺少座位数
- **WHEN** 用户选择"找乘客"类型但未填写座位数
- **THEN** 系统显示错误提示"请填写座位数"
- **THEN** 系统不提交表单

### Requirement: 字段格式验证
系统 SHALL 验证字段格式的有效性。

#### Scenario: 出发地目的地长度验证
- **WHEN** 用户输入的出发地或目的地少于 2 个字符
- **THEN** 系统显示错误提示"地点名称至少 2 个字符"

#### Scenario: 座位数范围验证
- **WHEN** 用户输入的座位数不在 1-8 范围内
- **THEN** 系统显示错误提示"座位数应在 1-8 之间"

#### Scenario: 价格范围验证
- **WHEN** 用户输入的价格超过 9999 元
- **THEN** 系统显示错误提示"价格不能超过 9999 元"

#### Scenario: 备注长度验证
- **WHEN** 用户输入的备注超过 200 字符
- **THEN** 系统显示错误提示"备注不能超过 200 字符"

### Requirement: 实时验证反馈
系统 SHALL 在用户失焦时进行实时验证并显示错误提示。

#### Scenario: 失焦时验证
- **WHEN** 用户在某个字段失焦
- **THEN** 系统立即验证该字段
- **THEN** 如果验证失败，在字段下方显示错误提示

#### Scenario: 修正错误后清除提示
- **WHEN** 用户修正了错误的字段
- **THEN** 系统清除该字段的错误提示

### Requirement: 提交表单
系统 SHALL 在验证通过后调用 API 提交表单数据。

#### Scenario: 成功提交
- **WHEN** 用户填写完整且有效的表单并点击发布按钮
- **THEN** 系统显示 loading 状态
- **THEN** 系统调用发布 API
- **THEN** API 返回成功后显示"发布成功"提示
- **THEN** 系统跳转到首页

#### Scenario: 提交失败
- **WHEN** API 返回错误
- **THEN** 系统显示错误提示信息
- **THEN** 系统保持在发布页面
- **THEN** 用户可以重新提交

### Requirement: 表单重置
系统 SHALL 在发布成功后清空表单数据。

#### Scenario: 发布成功后清空表单
- **WHEN** 发布成功并跳转到首页后
- **THEN** 如果用户返回发布页面，表单应为空白状态
